# Enforce the qmd query-path (tiered fidelity), and record the discovery path per run

ADR 0030 made qmd the **mandatory wiki/docs search layer** — but "mandatory" only ever bit in two places: the workspace *installs + indexes* qmd at setup, and (after ADR 0036) the orchestrator deterministically *re-embeds* after an ingest. The **read/query path was left to prose**: all six agent profiles say "qmd-first," but nothing mechanically stops an agent from `grep`-ing the wiki to discover pages, and `governance-classify.cjs` rates `grep`/`rg`/`Read` as **LOW → auto-allowed**, so a hand-scan sails through with zero friction. ADR 0030 also *deliberately rejected* a hard-stop-if-absent (its option D) and sanctioned an `index.md`-navigation fallback, so "qmd is mandatory" was, at query time, **voluntary compliance with a legal escape hatch** — not "always qmd." This ADR closes that gap without reintroducing the brittleness 0030 rejected.

## Design

- **Query-path is enforced where it can be, observed everywhere.** Enforcement is inherently *tiered* because only Claude Code has a live interception point (the `--permission-prompt-tool` governance MCP). We therefore split the guarantee: a **gate** on the tier that can be gated, and a portable **run-record signal** on every tier — the same shape ADR 0036 used for the embed outcome.
- **The gate (Claude tier) is grep-focused, with carve-outs.** The per-run governance gate soft-**denies** a raw full-text `grep`/`rg`/`find`-content over `.tcgstackflow/wiki/**` page *bodies* when qmd is available and **no qmd query has run this run** — returning a *"discover via wiki-search (qmd) first, or navigate index.md"* redirect (a `deny` with guidance, not a browser approval; this is methodology, not risk). **Always allowed:** reading/navigating `index.md`; the sanctioned `grep "^## \[" log.md` timeline read; and every `Read` (a Read is the "open the page qmd surfaced" step, not discovery). The gate watches Bash for a `qmd query|search|vsearch` call to flip its per-run `qmdSeen` flag, and treats qmd as absent once a `qmd --version` fails. **Fail-safe stays fail-*open* for flow:** if the gate cannot tell, it allows — a governance gate must never block legitimate work it doesn't understand (distinct from the risk gate's fail-*closed* stance, because a missed discovery-grep is a quality lapse, not a safety breach).
- **The observable signal (all tiers).** The `wiki-search` skill records the discovery path it took into the run record as a sibling of the `embed` block: `wiki_discovery: { path: qmd | index-fallback | none, reason?: declined|missing|stale|thin-results, queries: <n> }`. On Claude the gate corroborates it. The Cockpit renders a per-run **badge** beside the fidelity/embed badges (`discovery: qmd` / `index-fallback: missing`), so "did this run actually use qmd?" is auditable rather than assumed.
- **One sanctioned non-qmd discovery path.** Structured `index.md` + `[[wikilink]]` navigation is the **only** blessed non-qmd way to *discover* pages — the always-current fallback when qmd is declined, missing, or stale. **Raw full-text grep over wiki bodies is never the fallback.** This removes the footgun in `wiki-search/SKILL.md`'s old anti-pattern wording ("grepping … *when qmd is available*"), whose conditional implicitly licensed the exact wikilink-blind scan the layer exists to prevent.
- **The subjective escape hatch is tightened.** The skill's "if results look thin or stale, fall back to `index.md`" now requires a documented refinement attempt first (switch `query`↔`search`↔`vsearch`, or refine the query) before abandoning qmd — reusing the retrieval-budget discipline already in the skill.
- **Freshness backstop for non-orchestrated edits.** The deterministic re-embed (ADR 0036) covered only the orchestrated Ingester. The `post-merge` hook now also runs an incremental `qmd embed` (guarded by `qmd --version`) when a merge touched `wiki/` or `docs/`, so a manual edit or a PR that lands wiki changes outside the ingest flow does not leave a stale index serving the readers the fallback then tells them to distrust.
- **The convention becomes a checkable invariant.** `audit-workspace`/`lint-wiki` gain a detector that flags any wiki-recalling skill whose body lacks a `wiki-search`/qmd step, and any agent profile missing the `index.md` fallback clause — so future drift is caught, not discovered.

## Considered options

- **(A) Gate on Claude + portable run-record signal + tiered fidelity** — *chosen*. Uses the two mechanisms the project already trusts (the governance interception; a run-record outcome field à la ADR 0036), enforces where it can, and degrades *explicitly* via a badge on tiers it can't gate. Honors 0030's "no hard stop" while making the query-path non-voluntary in practice.
- **(B) A new dedicated PreToolUse hook, separate from governance** — rejected: a brand-new Claude-only mechanism to build and maintain, cleaner in theory (methodology ≠ risk) but with no portable story and duplicating the interception the governance MCP already performs.
- **(C) Hard-fail if qmd absent (revisit 0030's option D)** — rejected again: brittle; a workspace that hasn't installed qmd could not search the wiki at all, and it discards the Map-of-Content safety net 0003/0030 deliberately preserve.
- **(D) Observe-only, no live gate** — rejected as insufficient for the stated intent: it makes non-compliance *visible* but not *prevented*; kept as the (unavoidable) behavior for the non-Claude tiers, surfaced honestly by the badge.
- **(E) Gate `Read` of wiki pages too (block "Read-scan to discover")** — rejected as the default: high false-positive risk on the legitimate "open what qmd surfaced" step, for a rare abuse the retrieval-budget rule already discourages. The grep-focused gate targets the documented `grep-the-wiki` anti-pattern precisely.

## Consequences

- `governance-classify.cjs` gains a wiki-discovery classification and `governance-mcp.cjs` gains a per-run `qmdSeen`/`qmdAbsent` state + a `deny`-with-guidance path for the pre-qmd body-grep case. `Read`/`Grep`/`Bash` must route through the gate (not be blanket pre-allowed) for the tier to bite; the gate auto-allows them as before and only soft-denies the one case. The risk gate's fail-**closed** behavior for HIGH/CRITICAL is untouched; the new discovery gate is fail-**open**.
- The run record schema gains an optional `wiki_discovery` block (read/written by the `read.cjs` serializer alongside `embed`); the Cockpit projection and SPA gain a badge. Absent block → no badge (older runs unaffected).
- `wiki-search/SKILL.md` (anti-pattern wording + thin/stale gate), `grill-task`/`plan-task`/`review-diff` skill bodies, and the `refactorer` profile are updated; ADR 0006 gets a supersession note pointing at 0030/0036/0037; `CONTEXT.md`'s **Wiki search (qmd)** term is disambiguated (install-time vs query-time "mandatory") — done alongside this ADR.
- The `post-merge` hook gains a guarded incremental re-embed on wiki/docs-touching merges.
- **Amends ADR 0030**: 0030 stands (qmd mandatory + complement + auto-install + index.md fallback); this ADR adds the *query-time enforcement + observability* 0030 left as prose, and pins that `index.md` navigation is the sole sanctioned non-qmd discovery path.
